# 多个汉字的字模位置计算

## 1. "好"字
- GB2312编码：0xBAC3（区码：0xBA，位码：0xC3）
- 起始地址：0x71300
- 每字符字节数：32
- 区码偏移：0xBA - 0xA1 = 25
- 位码偏移：0xC3 - 0xA1 = 34
- 字符索引：(25 × 94) + 34 = 2350 + 34 = 2384
- 总偏移量：2384 × 32 = 76288
- 最终字模位置：0x71300 + 76288 = 0x71300 + 0x129C0 = 0x83CC0

## 2. "世"字
- GB2312编码：0xCAC0（区码：0xCA，位码：0xC0）
- 起始地址：0x71300
- 每字符字节数：32
- 区码偏移：0xCA - 0xA1 = 29
- 位码偏移：0xC0 - 0xA1 = 31
- 字符索引：(29 × 94) + 31 = 2726 + 31 = 2757
- 总偏移量：2757 × 32 = 88224
- 最终字模位置：0x71300 + 88224 = 0x71300 + 0x158A0 = 0x86BA0

## 3. "界"字

- 字符索引：(27 × 94) + 83 = 2538 + 83 = 2621
- 总偏移量：2621 × 32 = 83872
- 最终字模位置：0x71300 + 83872 = 0x71300 + 0x14860 = 0x85B60

## 验证方法
使用以下命令读取字模数据：
```
read_font_data 0x83CC0 32  # 读取"好"字的字模数据
read_font_data 0x86BA0 32  # 读取"世"字的字模数据
read_font_data 0x85B60 32  # 读取"界"字的字模数据
```

## 结论
所有汉字的字模位置都可以通过公式：
位置 = 起始地址 + ((区码 - 0xA1) * 94 + (位码 - 0xA1)) * 每字符字节数
计算得出，无需使用任何预设转换表。
